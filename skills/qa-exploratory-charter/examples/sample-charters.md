# 예시: qa:exploratory-charter 출력

이 파일은 LLM이 출력 톤·구조를 학습하는 few-shot. `skills/qa-risk-analysis/examples/sample-risk-report.md`의 "라운지 선물하기" 리스크 리포트를 입력으로 받았을 때 기대되는 차터.

---

## 탐색적 테스트 차터 — 라운지 선물하기

### 차터 요약

| # | 목표 | 연계 리스크 | 타임박스 |
|---|---|---|---|
| 1 | 결제 중단·재시도 상황에서 젬 이중 차감 가능성을 탐색한다 | Blocker — 결제 이중 차감 (리스크 #1) | 60분 |
| 2 | `enableLoungeGift` 플래그 off 전환 타이밍에서 진입점 잔존 여부를 탐색한다 | Major — 플래그 off 시 진입점 노출 (리스크 #2) | 45분 |
| 3 | iOS 권한 거부→설정 변경→앱 복귀 전이에서 선물 수신 후 콜 진입 충돌을 탐색한다 | Major — iOS 권한 거부 시 크래시 (리스크 #3) | 45분 |

---

### 차터 1: 결제 중단·재시도 탐색

**목표:** 결제 진행 중 네트워크 끊김·백그라운드 전환·버튼 연타 조합 상황에서 젬 이중 차감 없이 정합성이 유지되는지 탐색한다.

**범위:** 선물 구매 시작 ~ 결제 완료/실패 화면까지의 전 구간; Charles/네트워크 조작으로 응답 지연·중단 시뮬레이션; 잔액·히스토리 서버 검증.

**비범위:** 정상 결제 성공 플로우(스크립트 TC로 커버), 수신자 측 경험.

**시작 지점:** 젬 잔액 확인 가능한 테스트 계정 로그인 상태, `enableLoungeGift` 플래그 on, Charles 프록시 연결 완료.

**타임박스:** 60분

**기록 항목:** 이중 차감 발생 여부(잔액·히스토리 비교), 재시도 중 중복 결제 팝업 노출 여부, 예상 밖 에러 코드·토스트, 후속 TC 후보(재시도 경계 값 고정 케이스).

---

### 차터 2: `enableLoungeGift` 플래그 토글 타이밍 탐색

**목표:** 사용자가 선물하기 플로우 진행 중 `enableLoungeGift` 플래그가 off로 전환될 때 진입점 잔존·딥링크 직접 접근 차단 동작을 탐색한다.

**범위:** 플래그 on → 선물하기 버튼 진입 → 플래그 off 전환(Remote Config 갱신) → 화면 재진입 시 버튼/딥링크 노출 여부; 탭 간 이동 후 복귀 시 상태 갱신 확인.

**비범위:** 플래그 최초 off 상태의 정상 진입 차단(스크립트 TC로 커버).

**시작 지점:** 테스트 Remote Config 환경, 플래그 실시간 토글 권한 보유, 라운지 탭 진입 상태.

**타임박스:** 45분

**기록 항목:** 플래그 전환 후 버튼이 사라지기까지 걸리는 시간, 딥링크 직접 호출 시 도달 가능 화면, 탭 포커스 이동 후 갱신 여부, 후속 TC 후보(플래그 전환 지연 허용 임계값).

---

### 차터 3: iOS 권한 상태 전이 탐색

**목표:** iOS 마이크·카메라 권한 거부 → 설정 앱에서 허용 변경 → 앱 복귀 후 선물 수신 콜 진입 전이 경로에서 크래시·비정상 상태를 탐색한다.

**범위:** 권한 거부 상태에서 선물 수신 알림 수신 → 콜 진입 시도; 설정 변경 후 앱 포그라운드 복귀; 권한 반복 토글(거부→허용→거부) 후 콜 재진입.

**비범위:** Android 동일 케이스(별도 세션 권장), 정상 권한 허용 상태의 콜 플로우(스크립트 TC로 커버).

**시작 지점:** iOS 테스트 기기(실기기 권장), 설정 > 앱 권한 거부 설정 완료, 선물 발신 가능한 보조 계정 준비.

**타임박스:** 45분

**기록 항목:** 크래시 발생 여부 및 재현 빈도, 권한 변경 후 앱 재기동 없이 상태가 갱신되는지 여부, 비정상 UI(권한 요청 재노출·블랙 화면), 후속 TC 후보(거부→허용 전환 후 콜 진입 고정 케이스).

---

> 발견한 이슈는 `qa:bug-report`로 리포트하고, 반복 가치가 있는 발견은 `qa:generate-tc`로 정식 TC화하세요.
