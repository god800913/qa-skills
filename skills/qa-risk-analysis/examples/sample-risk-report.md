# 예시: qa:risk-analysis 출력

이 파일은 LLM이 출력 톤·구조를 학습하는 few-shot. "라운지 선물하기" 기능 PRD를 입력으로 받았을 때 기대되는 리포트.

---

## 리스크 분석 — 라운지 선물하기

### 요약
결제 이중 차감(Blocker)이 최고 등급 — 결제·수익화 영역을 최우선 집중하고, Remote Config 플래그 off 케이스와 iOS 권한 거부 플로우를 병행 검증 필요.

### 리스크 매트릭스

| # | 영역 | 리스크 | 영향도 | 발생가능성 | 등급 | 권장 테스트 포커스 |
|---|---|---|---|---|---|---|
| 1 | 결제·수익화 | 선물 구매 시 젬 이중 차감 (네트워크 재시도 + 서버 중복 처리 경쟁) | Blocker | 높음 | **Blocker** | 결제 재시도 시나리오, 서버 멱등성 확인, 잔액 정합성 검증 |
| 2 | Remote Config·롤아웃·실험 | `enableLoungeGift` 플래그 off 시 선물하기 진입점(버튼/딥링크)이 잔존할 가능성 | Major | 중간 | **Major** | 플래그 off 상태에서 진입점 노출 여부 전수 확인, 딥링크 직접 접근 차단 검증 |
| 3 | 크로스플랫폼·OS 분기 | iOS 마이크/카메라 권한 거부 상태에서 선물 발송 후 수신자 콜 진입 시 앱 크래시 | Major | 중간 | **Major** | iOS 권한 거부 → 선물 수신 → 콜 진입 전체 플로우, Android 동일 케이스 비교 |
| 4 | UI·카피 | 다크모드에서 선물 수신 알림 카피 대비(contrast)가 디자인 기준 미달 가능성 | Minor | 낮음 | **Minor** | 다크모드 기기에서 알림·수신 화면 시각적 확인 |

### 가정

1. `enableLoungeGift` Remote Config 플래그가 존재하고 기본값이 off라고 가정. — **PM 확인 필요**: 플래그 이름과 기본값, 롤아웃 대상(전체/일부 사용자) 확인 요청.
2. 선물 수신 후 콜 연결 플로우가 기존 라운지 매치 콜과 동일한 권한 요청 로직을 사용한다고 가정. — **PM/개발 확인 필요**: 별도 권한 요청 분기가 있는지 확인 요청.
