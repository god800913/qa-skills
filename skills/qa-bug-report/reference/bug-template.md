# 버그 리포트 표준 양식

## 제목 규칙

```
[영역] 증상 — 발생 조건
```

- **영역**: 기능 도메인 (예: `라운지 추천`, `선물하기`, `매칭`, `신고`)
- **증상**: 관찰된 실제 동작 (명사/동사구, 20자 내외)
- **발생 조건**: 재현에 필요한 핵심 전제 (환경·상태·트리거)

### 좋은 제목 vs 나쁜 제목

| 나쁜 제목 | 좋은 제목 |
|---|---|
| `버그 발견` | `[라운지 추천] 딥링크 진입 시 스크롤 최상단 고정 — iOS 17.5 앱 ver118` |
| `선물 안 됨` | `[선물하기] 결제 중 백그라운드 전환 후 재진입 시 이중 차감 — enableLoungeGift ON` |

## 필드 정의

| 필드 | 필수 | 설명 |
|---|---|---|
| **제목** | ✅ | `[영역] 증상 — 조건` 형식 |
| **환경** | ✅ | 기기·OS 버전·앱 버전 (예: iPhone 14 / iOS 17.5 / ver118) |
| **사전 조건** | ✅ | 계정 상태, Remote Config 플래그 값, 로그인 여부 등 |
| **재현 단계** | ✅ | 번호 매긴 단계 — TC Test Step 기반, 재현에 필요한 세부 보충 |
| **기대 결과** | ✅ | TC Expected Result 기반 |
| **실제 결과** | ✅ | 사용자 보고 그대로 — 추정 금지 |
| **심각도 + 근거** | ✅ | Blocker/Major/Minor, risk-taxonomy 영역 기준 명시 |
| **재현 빈도** | ✅ | N회 중 M회 (예: 5/5, 3/5) |
| **관련 TC_ID** | ✅ | Fail/Block된 TC_ID (예: `1-4`) |
| **첨부** | 권장 | 스크린 레코딩, 로그, 스크린샷 파일명 또는 링크 |

## 심각도 판정 기준

| 심각도 | 기준 | risk-taxonomy 영역 예시 |
|---|---|---|
| **Blocker** | 핵심 플로우 완전 차단, 데이터 손실·이중 과금 가능성 | 결제, 신고, 차단, 핵심 CTA |
| **Major** | 핵심 플로우에 인접하나 우회 가능, 사용성 심각 저하 | 추천 노출, 딥링크, 권한 요청 |
| **Minor** | 우회 가능, 시각적 오류, 엣지 케이스 | 텍스트 오탈자, 레이아웃 미세 틀어짐 |

결제·신고·차단 관련 영역은 재현 빈도가 낮더라도 **Blocker 이상으로 상향 검토**.
