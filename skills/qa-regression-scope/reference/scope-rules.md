# 회귀 범위 판정 규칙 (scope-rules)

`risk-taxonomy.md`의 영역명을 기준으로 Required / Optional / Skipped 를 판정한다.

## 강제 포함 (Required)

| 조건 | 예시 영역 |
|---|---|
| 이번 변경이 직접 수정한 코드 경로가 속한 영역 | 결제·수익화, 매치·콜·실시간 등 변경 당사자 탭/섹션 |
| 기본 영향도가 Blocker인 영역 (변경 여부 무관) | 핵심 사용자 플로우, 결제·수익화, 신고·차단·모더레이션 |
| 변경 영역과 공유 컴포넌트(결제 SDK, 인증 토큰, 공통 라우터 등)를 사용하는 영역 | 결제·수익화 ↔ 매치·콜·실시간 (젬 소모 공유 시) |
| 이전 두 릴리즈에서 해당 영역에 버그가 발생한 이력이 있는 경우 | Remote Config·롤아웃·실험 (플래그 오작동 반복 시) |

## 선택 포함 (Optional)

| 조건 | 예시 영역 |
|---|---|
| 변경 영역과 간접 의존(API 응답 공유, 화면 전환 연결)이 있는 영역 | UI·카피 (선물 수신 화면 레이아웃 간접 영향) |
| 기본 영향도 Major이고 코드 경로가 인접하지만 직접 변경은 없는 경우 | 크로스플랫폼·OS 분기, 마이그레이션·하위호환 |

## 안전 생략 (Skipped)

| 조건 | 잔여 리스크 수준 |
|---|---|
| 독립 모듈 — 변경 영역과 코드·데이터 공유 없음 | 낮음 |
| 최근 N릴리즈(N≥3) 동안 무변경·무결함 영역 | 낮음 |
| 기본 영향도 Minor이고 이번 변경과 접점 없음 | 낮음~무시 가능 |

> **원칙**: Skipped 항목은 반드시 생략 근거와 잔여 리스크를 리포트에 명시한다.
> 근거 없는 생략은 인정하지 않는다.

## Open Questions
