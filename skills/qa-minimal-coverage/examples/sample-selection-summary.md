# 예시: qa:minimal-coverage 출력

이 파일은 LLM이 출력 톤·구조를 학습하는 few-shot. "라운지 선물하기" 탭(28 TC)에서 `--max-cases 10` 옵션으로 실행한 결과 보고 모양.

---

## 최소 커버리지 선택 결과 — TabA (라운지 선물하기)

### 선택 요약

- **선택: 10 / 전체: 28** (Next Best 5건 · Excluded 13건)
- **커버된 리스크 태그:** 결제, 구매, 환불, 매치, 콜, remote-config, os:iOS, os:Android (8종)
- **출력 경로:** `/tmp/lounge-gift-minimal.xlsx`

```
강제 포함(P1·고위험 키워드) 후보 11건 → 10건 선택 (risk_score 내림차순, 동점 시 원본 순서)
greedy 커버 확대                     →  0건 (max-cases=10 도달로 진입 전 종료)
------------------------------------------------------------------
Total: 10 / 28 TC   (잔여 18건 = Next Best 5건 + Excluded 13건)
```

### Forced-Overflow 경고

> **강제 포함 대상 1건이 max-cases 초과로 제외되었습니다.**
> TC_ID `SH-07` (P2, 매치 콜 실패 복구) — 강제 포함 후보 11건 중 risk_score 최하위(3.0)로 밀려 한도(10건)에 들지 못했습니다.
> 점수가 높아 Next Best 1순위에 위치합니다. 한도에 밀린 강제 대상은 Next Best·Excluded 시트 모두 `강제 대상` 컬럼에 "Y"로 표시됩니다. 직접 실행 여부를 검토하세요.

### 잔여 리스크 경고

> 제외된 18건의 리스크 태그는 모두 선택 세트가 이미 커버합니다 (미커버 신규 태그 없음).
> 단, 강제 포함 대상 `SH-07`이 한도 때문에 빠진 것 자체가 실질 잔여 리스크입니다 — 매치·콜은 Blocker 영역(`reference/risk-taxonomy.md`)이므로 시간 여유가 생기면 가장 먼저 실행하세요.

### 가정 (Assumptions)

- `score = risk_score + coverage_gain - execution_cost - redundancy_penalty`
- 강제 포함: Priority P1 또는 고위험 키워드 (결제·신고·매치·콜·권한·Remote Config 등)
- `max-cases=10` 제한으로 강제 포함 대상 일부 제외 — 잔여 리스크 확인 필요
