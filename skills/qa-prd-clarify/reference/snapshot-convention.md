# PRD 스냅샷 규약 (snapshot-convention)

> `qa:prd-clarify` · `qa:generate-tc`가 fetch 성공 시 저장하고, `qa:prd-diff`가 변경분 분석에 소비한다.
> 핵심 원칙: **스냅샷은 fetch 원문의 보존이지 재서술이 아니다.** LLM이 구조를 다듬은 스냅샷은 prd-diff에서 실제 스펙 변경과 구분이 안 되는 노이즈를 만든다.

## 1. 경로·파일명

- 기본 경로: `./prd-snapshots/<슬러그>-<YYYYMMDD>.md` (디렉토리 없으면 생성, 사용자가 다른 경로 지정 시 그에 따름)
- 날짜는 `date '+%Y%m%d'`로 구한다 — 암산 금지.
- 슬러그는 Notion 페이지 제목(없으면 PRD 첫 H1)에서 생성:
  - 한글은 그대로 유지, 영문은 소문자화
  - 공백·특수문자(`/ \ : * ? " < > | [ ]` 등)는 하이픈으로 치환
  - 연속 하이픈은 1개로 축약, 양 끝 하이픈 제거
  - 예: `KR - KMC 본인인증 도입` → `kr-kmc-본인인증-도입`

## 2. 파일명 충돌 (같은 날 재실행)

같은 경로의 파일이 이미 있으면:
1. 새로 fetch한 본문과 기존 파일 본문이 **동일** → 저장 skip, "기존 스냅샷과 동일" 알림
2. **다르면** → `<슬러그>-<YYYYMMDD>-2.md`, `-3.md` … 새 파일로 저장 (기존 파일 덮어쓰기 금지)

## 3. 본문 형식

```markdown
---
prd_url: <Notion URL>
fetched_at: <YYYY-MM-DD>
sources:
  - main: <메인 페이지 제목>
  - linked: <1-depth 포함한 하위 페이지 제목>  # 없으면 생략
  - excluded: <미포함 소스와 사유>             # 없으면 생략
---

<fetch 원문>
```

- **fetch 원문은 verbatim 보존** — 요약·재구성·섹션 재배치·문장 다듬기 금지. fetch 도구가 반환한 마크다운을 그대로 둔다.
- 1-depth로 follow한 하위 페이지는 메인 본문 뒤에 `\n\n---\n## [linked] <페이지 제목>\n\n` 구분자로 이어붙인다.
- Notion 서식 중 마크다운으로 손실되는 정보는 주석 표기로 고정:
  - 취소선: `~~텍스트~~` 그대로 유지 (삭제된 스펙 표시일 수 있음 — diff에 의미 있음)
  - 색상·하이라이트가 fetch 결과에 안 들어오면 무시 (주석 추가하지 않음 — 추정 금지)

## 4. prd-diff 소비 시 주의

- 메타헤더(`---` 블록)는 비교 대상에서 제외.
- 구버전 스냅샷이 이 규약 이전(LLM 재서술본)일 수 있다 — 문장 표현·순서·헤더 구조 차이는 **형식 노이즈로 분류하고 변경점에서 제외**, 의미가 달라진 항목만 변경점으로 보고.
